# libs/widgets 模块介绍

## 这个模块是做什么的

`libs/widgets` 是多利机器人的屏幕小组件服务，当前主要负责在眼睛 LCD 上显示时钟和定时器这类“工具型界面”。

## 小白可以怎么理解它

平时眼睛 LCD 是 `modules/eyeEngine` 用来显示表情的，但有时你希望它临时显示：

- 当前时间
- 定时器倒计时
- 报时或提醒信息

这种不是“眼睛表情”，而是“工具界面”的内容，就由 `widgets` 处理。

## 它主要负责什么

- 独立运行时钟和定时器 Widget 服务
- 订阅 `cmd.widget.*` 命令
- 发布 `event.widget.*` 和 `status.widget.*` 事件
- 抢占并释放 LCD 使用权
- 与 `modules/eyeEngine` 做跨进程 LCD 互斥协调

从 `src/widget_service.cpp` 可以确认它是一个独立服务，支持：

- 时钟显示
- 定时器启动、暂停、恢复、停止
- LCD 文件锁互斥
- ZMQ 命令与状态发布

## 输入与输出

### 输入

- Widget 命令，例如显示时钟、启动定时器
- 配置文件
- 来自主控的交互请求

### 输出

- LCD 上的时钟或定时器画面
- LCD 抢占/释放事件
- 定时器状态事件和状态快照

## 它依赖谁、又被谁依赖

### 它依赖谁

- 眼屏 LCD 相关底层能力
- ZeroMQ
- 文件锁机制

### 谁会用它

- `modules/doly`：通过 Widget 管理器控制时钟和定时器
- `modules/eyeEngine`：与其协调 LCD 的使用权

## 一个典型调用链

1. 主控发出 `cmd.widget.clock.show` 或 `cmd.widget.timer.start`
2. `widgets` 服务接收命令并尝试获取 LCD 锁
3. 获取成功后在屏幕上渲染时钟/定时器
4. 发布 `event.widget.lcd_acquired` 等事件
5. 结束后释放 LCD，并通知 `eyeEngine` 恢复眼睛显示

## 什么时候你应该看这个模块

- 时钟/定时器界面不显示
- Widget 和眼睛动画抢屏
- 定时器状态事件不对
- 想新增新的屏幕工具组件

- `docs/arch/00_设计文档.md`

## 一句话总结

这是多利眼屏上的“实用工具层”，和 `eyeEngine` 一起共享同一块 LCD。
