# libs/nlu/doly-nlpjs 模块介绍

## 这个模块是做什么的

`libs/nlu/doly-nlpjs` 是多利机器人的自然语言理解服务。它负责把文本形式的用户输入解析成“意图”，例如问候、查询、控制类命令等，方便上层主控继续做动作决策。

## 小白可以怎么理解它

语音识别只能把人说的话转成文字，但机器人还需要知道“这句话想干什么”。

`doly-nlpjs` 做的就是这一步：

- 输入：一句中文文本
- 输出：一个意图名 + 置信度 + 实体信息

## 它主要负责什么

- 从语料库加载训练数据
- 训练 NLP.js 模型并保存为 `model.nlp`
- 通过 HTTP 提供文本解析服务
- 返回意图、置信度、实体和可选回复
- 支持健康检查、统计信息、模型热重载

从 `server.js` 可以确认它提供了这些接口：

- `POST /parse`
- `POST /process`
- `GET /health`
- `GET /stats`
- `POST /reload`

从 `train.js` 可以确认它会读取中文语料并训练模型。

## 输入与输出

### 输入

- 用户文本
- 训练语料 JSON 文件

### 输出

- 意图名
- 置信度
- 实体信息
- 可选应答文本

## 它依赖谁、又被谁依赖

### 它依赖谁

- Node.js
- `node-nlp`
- 语料库配置

### 谁会用它

- `modules/doly` 或上层语音/对话链路
- 需要把文本命令转成系统动作的业务模块

## 一个典型调用链

1. 用户说话，前面的语音识别模块把语音转成文本
2. 文本发到 `doly-nlpjs`
3. 服务返回意图和置信度
4. 主控根据意图映射到动画、行为或系统指令

## 什么时候你应该看这个模块

- 文本能拿到，但机器人理解错意思
- 新增了语音命令，却无法识别
- 训练模型后效果不稳定
- 需要给系统增加一套新的中文语料

## 建议先看哪些文件

- `package.json`
- `server.js`
- `train.js`
- `model.nlp`
- `scripts/`

## 备注

这个模块当前没有独立文档目录，本文主要依据 `package.json`、`server.js` 和 `train.js` 的真实实现整理。它是“文本理解”环节，不负责语音转文字，也不直接负责动作执行。
