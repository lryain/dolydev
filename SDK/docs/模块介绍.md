# SDK 模块介绍

## 这个模块是做什么的

`SDK` 是多利机器人对开发者开放的统一接口层。它把底层硬件能力、Python/C++ 示例和文档整理成更容易上手的形式，方便开发者快速调用机器人能力。

## 小白可以怎么理解它

如果你不是要改系统内部架构，而是想“写点代码让多利动起来、亮起来、说起来”，你最应该先看的通常就是 `SDK`。

它相当于“面向开发者的使用入口”。

## 它主要负责什么

- 暴露机械臂、底盘、眼睛、LED、ToF、TTS 等基础 API
- 提供 C++/Python 头文件和示例代码
- 提供文档站入口和各模块参考文档
- 帮助开发者绕过底层细节，更快完成集成

`SDK/include/` 里能看到一整套清晰的接口文件，例如：

- `ArmControl.h`
- `DriveControl.h`
- `EyeControl.h`
- `LcdControl.h`
- `TofControl.h`
- `TtsControl.h`

同时 `SDK/docs/` 下已经整理了 Python SDK 文档与示例。

## 输入与输出

### 输入

- 开发者的 C++/Python 调用
- 设备控制参数和监听器

### 输出

- 统一的机器人能力调用结果
- 示例代码和 API 文档

## 它依赖谁、又被谁依赖

### 它依赖谁

- `libs/Doly` 这类底层能力库
- 示例程序和文档资源

### 谁会用它

- 外部开发者
- 编写测试脚本、演示脚本和教学代码的人
- 需要快速调用机器人基础能力的上层业务代码

## 一个典型使用方式

1. 查阅 `SDK/docs`
2. 找到需要的能力接口或 Python 示例
3. 按示例导入模块或头文件
4. 初始化模块并发送控制命令
5. 监听回调或读取状态

## 什么时候你应该看这个模块

- 你是第一次接触多利项目
- 想用最短时间验证硬件能力是否可用
- 想找官方/项目内已有的调用示例
- 想确认某个底层能力对外暴露成什么 API

## 建议先看哪些文件

- `README.md`
- `docs/`
- `include/`
- `examples/`

## 一句话总结

`SDK` 是多利项目对开发者最友好的入口层，适合先学会“怎么用”，再回头深入内部模块。
