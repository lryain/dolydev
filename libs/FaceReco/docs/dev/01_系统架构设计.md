## 1. 概述

### 1.1 目标

将 FaceReco 模块重构为独立运行的**视觉服务模块**（Vision Service），实现：

1. **模块化运行**: 支持空载模式、单功能模式（推流/检测/识别）和完整模式
2. **ZeroMQ 通信**: 统一的消息总线接口，支持命令、事件和查询
3. **人脸管理**: 完善的人脸数据 CRUD 功能
4. **功能扩展**: 拍照、录视频、实时推流到 LCD
5. **行为联动**: 与 Daemon 的 FaceRecoManager 协作，触发机器人行为

### 2.2 模块类型定义

```cpp
// include/doly/vision/module_types.hpp

namespace doly::vision {

// 运行模式
enum class RunMode {
    IDLE,           // 空载模式（不加载任何功能模块）
    STREAM_ONLY,    // 仅视频推流
    DETECT_ONLY,    // 仅人脸检测（不识别）
    DETECT_TRACK,   // 人脸检测 + 跟踪
    FULL,           // 完整模式（检测 + 识别 + 推流）
    CUSTOM          // 自定义模块组合
};

// 模块状态
enum class ModuleState {
    UNLOADED,
    LOADING,
    RUNNING,
    PAUSED,
    ERROR
};

// 可加载的功能模块
enum class ModuleType {
    CAMERA_CAPTURE,     // 摄像头采集
    FACE_DETECTION,     // 人脸检测
    FACE_RECOGNITION,   // 人脸识别
    FACE_TRACKING,      // 人脸跟踪
    LIVENESS_DETECTION, // 活体检测
    VIDEO_STREAM,       // 视频流发布
    PHOTO_CAPTURE,      // 拍照
    VIDEO_RECORD        // 录像
};

}  // namespace doly::vision
```

## 3. ZeroMQ 通信协议

### 3.1 话题规划

| 话题 | 方向 | 描述 |
|------|------|------|
| **命令话题 (Commands)** | | |
| `cmd.vision.mode` | IN | 设置运行模式 |
| `cmd.vision.enable` | IN | 启用/禁用服务 |
| `cmd.vision.module.load` | IN | 加载指定模块 |
| `cmd.vision.module.unload` | IN | 卸载指定模块 |
| `cmd.vision.config` | IN | 修改配置参数 |
| `cmd.vision.face.register` | IN | 注册人脸 |
| `cmd.vision.face.update` | IN | 更新人脸标签 |
| `cmd.vision.face.delete` | IN | 删除人脸 |
| `cmd.vision.face.query` | IN | 查询人脸数据 |
| `cmd.vision.capture.photo` | IN | 拍照 |
| `cmd.vision.capture.video.start` | IN | 开始录像 |
| `cmd.vision.capture.video.stop` | IN | 停止录像 |
| `cmd.vision.stream` | IN | 控制视频流 |
| **事件话题 (Events)** | | |
| `event.vision.face` | OUT | 人脸检测事件（实时坐标） |
| `event.vision.face.recognized` | OUT | 人脸识别成功事件 |
| `event.vision.face.lost` | OUT | 人脸消失事件 |
| `event.vision.face.new` | OUT | 新人脸出现事件 |
| `event.vision.capture.complete` | OUT | 拍照/录像完成事件 |
| `event.vision.module.state` | OUT | 模块状态变更事件 |
| **状态话题 (Status)** | | |
| `status.vision.state` | OUT | 服务状态快照 |
| `status.vision.faces` | OUT | 当前人脸列表快照 |
| **查询话题 (Queries)** | | |
| `query.vision.face.list` | REQ/REP | 查询人脸数据库 |
| `query.vision.face.position` | REQ/REP | 查询当前人脸位置 |
| `query.vision.config` | REQ/REP | 查询配置参数 |

### 3.2 消息格式

所有消息使用 JSON 格式，遵循统一信封结构：

```json
{
  "ts": 1738972800000,
  "src": "vision_service",
  "request_id": "uuid-xxxx",
  "data": {
    // 具体消息内容
  }
}
```

### 3.3 关键命令示例

#### 3.3.1 设置运行模式

```json
// Topic: cmd.vision.mode
{
  "mode": "FULL",  // IDLE, STREAM_ONLY, DETECT_ONLY, DETECT_TRACK, FULL, CUSTOM
  "modules": ["FACE_DETECTION", "FACE_RECOGNITION"]  // CUSTOM 模式时指定
}
```

#### 3.3.2 注册人脸

```json
// Topic: cmd.vision.face.register
{
  "method": "current",  // current=使用当前检测到的人脸, file=使用指定图片
  "name": "张三",
  "face_id": 123,  // 如果 method=current，指定当前跟踪的人脸 ID
  "file_path": "/path/to/image.jpg"  // 如果 method=file
}
```

#### 3.3.3 更新人脸标签

```json
// Topic: cmd.vision.face.update
{
  "id": "face_001",  // 人脸数据库 ID
  "name": "新名字",
  "metadata": {
    "relation": "朋友",
    "notes": "备注信息"
  }
}
```

#### 3.3.4 拍照

```json
// Topic: cmd.vision.capture.photo
{
  "save_path": "/home/pi/photos/",
  "filename": "photo_001.jpg",  // 可选，自动生成时间戳文件名
  "include_annotations": true,  // 是否包含人脸框标注
  "quality": 95
}
```

#### 3.3.5 人脸识别事件

```json
// Topic: event.vision.face.recognized
{
  "event": "face_recognized",
  "face_id": 123,
  "name": "张三",
  "confidence": 0.95,
  "liveness": true,
  "bbox": {"x1": 100, "y1": 80, "x2": 200, "y2": 180},
  "normalized": {"x": 0.45, "y": 0.52},
  "timestamp_ms": 1738972800000
}
```

## 5. 性能考量

### 5.1 LCD 推流性能评估

| 参数 | 目标值 | 备注 |
|------|--------|------|
| 分辨率 | 240x240 | LCD 原生分辨率 |
| 帧率 | 15-20 FPS | 平衡性能与流畅度 |
| 延迟 | < 100ms | 实时交互体验 |
| CPU 占用 | < 30% | 为其他模块保留资源 |

### 5.2 优化策略

1. **帧率控制**: 可配置的帧跳过机制
2. **分辨率适配**: 推流前降采样到 LCD 分辨率
3. **编码优化**: 使用高效的内存共享传输
4. **异步处理**: 推流与检测/识别分离
