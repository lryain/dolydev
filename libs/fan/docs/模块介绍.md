# libs/fan 模块介绍

## 这个模块是做什么的

`libs/fan` 是多利机器人的风扇温控服务，负责持续监控系统温度，并根据温度区间自动调节风扇转速，保护设备稳定运行。

## 小白可以怎么理解它

这个模块不是直接参与交互、动画或动作的，但它对整机稳定性很重要。

只要机器人长时间运行、播放动画、做视觉推理或执行复杂任务，CPU 温度就会上升，这时 `fan` 模块负责自动“散热”。

## 它主要负责什么

- 读取 CPU 温度
- 根据配置文件计算风扇 PWM 输出
- 通过 PCA9685 调节风扇速度
- 以 systemd 服务方式常驻运行
- 支持重载配置、查看状态和日志

## 输入与输出

### 输入

- CPU 温度
- `config/fan_config.yaml` 配置参数
- systemd/service 管理命令

### 输出

- 风扇 PWM 控制值
- 服务日志、运行状态

## 它依赖谁、又被谁依赖

### 它依赖谁

- 系统温度文件
- PCA9685 PWM 控制能力
- systemd 管理

### 谁会用它

- 它通常独立运行，不是被业务逻辑频繁调用的模块
- 主控模块更多是“受益于它维持稳定”，而不是直接操控它

## 一个典型工作流程

1. 服务启动后周期性读取 CPU 温度
2. 根据温度阈值和线性区间计算目标 PWM
3. 调整风扇转速
4. 记录状态变化与运行日志

## 什么时候你应该看这个模块

- 设备发热严重但风扇不转
- 风扇一直高速转，怀疑配置不合理
- 你要调整温控曲线或部署方式
