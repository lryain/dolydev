# libs/tof 模块介绍

## 这个模块是做什么的

`libs/tof` 是多利机器人的 ToF 测距模块，负责读取左右 ToF 传感器的数据，并把距离、环境光和障碍信息提供给上层使用。

## 小白可以怎么理解它

这个模块让机器人知道“前方近不近”“左右是否平衡”“附近有没有障碍物”。

它对应的是感知能力，不是执行能力。

## 它主要负责什么

- 初始化和配置 ToF 传感器
- 读取左右距离值
- 可选读取环境光数据
- 根据阈值判断是否存在障碍
- 通过服务方式对外提供查询结果

从 `tof_service.py` 可以确认，它是一个被动式 ZMQ 服务，负责：

- 加载配置
- 初始化硬件或 mock 模式
- 缓存最近一次传感器快照
- 返回左右距离、最小距离、平衡误差、障碍物判定等信息

## 输入与输出

### 输入

- 传感器硬件数据
- 配置文件
- 外部查询请求

### 输出

- 左右 ToF 距离值
- 障碍物是否存在
- 平衡误差
- 可选环境光数据

## 它依赖谁、又被谁依赖

### 它依赖谁

- VL6180X 等 ToF 相关硬件和库
- 配置文件和校准文件

### 谁会用它

- `modules/doly`：做探索、避障、交互判断
- 测试脚本和调试脚本

## 一个典型调用链

1. ToF 服务启动并加载配置
2. 初始化左右传感器与校准参数
3. 外部模块请求最新距离数据
4. 服务返回左右距离和障碍判定
5. 主控根据结果决定避障、停下或转向

## 什么时候你应该看这个模块

- 机器人避障异常
- 左右测距不一致，怀疑传感器偏移
- ToF 服务启动失败
- 需要调阈值、做校准或接入 mock 测试
