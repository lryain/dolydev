# libs/FaceReco 模块介绍

## 这个模块是做什么的

`libs/FaceReco` 是多利机器人的视觉识别模块，负责基于摄像头图像做人脸检测、人脸识别和活体判断，并把识别结果提供给系统其他部分使用。

## 小白可以怎么理解它

当机器人“看见你是谁”，或者判断当前镜头前有没有人、是不是活人，这部分能力主要来自 `FaceReco`。

它不是简单地拍照，而是负责一整条视觉识别链路。

## 它主要负责什么

- 摄像头图像中的人脸检测
- 人脸识别与身份匹配
- 活体检测，降低照片/屏幕欺骗风险
- 作为独立视觉服务对外提供事件和查询能力
- 为主控模块提供“谁出现了、谁离开了、当前位置在哪”的视觉信息

从现有文档可知，这个模块已经朝“独立视觉服务”方向建设，并规划了：

- 模块化运行模式
- ZeroMQ 事件总线接入
- 人脸数据库管理
- 与 Daemon 的行为联动

## 输入与输出

### 输入

- 摄像头视频流或图像帧
- 服务命令，例如启停、模式切换、注册人脸、查询人脸

### 输出

- 人脸检测结果
- 人脸识别结果
- 活体判断结果
- 面向主控和其它模块的视觉事件

## 它依赖谁、又被谁依赖

### 它依赖谁

- 视觉模型与推理库
- 摄像头输入
- ZeroMQ 总线或服务接口

### 谁会用它

- `modules/doly`：主控通过 `FaceRecoManager` 使用视觉事件
- 可能的显示/交互模块：根据识别结果决定表现行为

## 一个典型调用链

1. 摄像头持续提供图像
2. `FaceReco` 执行人脸检测、识别和活体判断
3. 结果通过事件或查询接口提供给上层
4. `modules/doly` 根据识别结果触发欢迎、互动或状态变化

## 什么时候你应该看这个模块

- 机器人看不到人脸
- 能检测到脸，但识别不稳定
- 活体判断误判较多
- 主控收不到视觉事件
