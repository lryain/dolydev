# Doly 手动更新脚本说明

本文对应的脚本位于 [scripts/update_and_redeploy.sh](../../scripts/update_and_redeploy.sh)。

## 背景

1. 多利代码仓库 `/home/pi/dolydev` 会持续更新。
2. 多利的功能模块主要以服务形式运行。
3. 每次更新后都需要重新部署相关服务。

## 这次更新脚本的职责

1. 将仓库更新到远端最新代码。
2. 当本地存在未提交修改、分支分叉或需要合并时，主动提示用户选择 `merge` 或 `reset`。
3. 按顺序重新编译并部署各个服务模块。

## 实际会重新部署的模块

1. [libs/audio_player/scripts/manage_audio_player_service.sh](../../libs/audio_player/scripts/manage_audio_player_service.sh)
2. [libs/drive/scripts/manage_service.sh](../../libs/drive/scripts/manage_service.sh)
3. [modules/eyeEngine/scripts/manage_service.sh](../../modules/eyeEngine/scripts/manage_service.sh)
4. [libs/fan/scripts/manage_service.sh](../../libs/fan/scripts/manage_service.sh)
5. [libs/nlu/doly-nlpjs/scripts/manage_service.sh](../../libs/nlu/doly-nlpjs/scripts/manage_service.sh)
6. [libs/serial/scripts/manage_serial_service.sh](../../libs/serial/scripts/manage_serial_service.sh)
7. [libs/tts/edge-tts/scripts/manage_service.sh](../../libs/tts/edge-tts/scripts/manage_service.sh)
8. [libs/widgets/scripts/manage_service.sh](../../libs/widgets/scripts/manage_service.sh)

## 使用方法

1. 进入仓库根目录。
2. 执行 `bash scripts/update_and_redeploy.sh`。
3. 如果脚本检测到本地改动或分支分叉，按提示选择：
   - `merge`：保留本地修改并合并远端更新。
   - `reset`：放弃本地修改，强制对齐远端最新代码。
   - `abort`：退出，不做任何修改。

然后脚本会自动拉取最新代码，并自动编译部署！

### 可选：仅部署单个模块

现在 `scripts/update_and_redeploy.sh` 支持传入模块名参数来限制仅部署指定模块。例如只部署 `audio_player`：

```bash
./scripts/update_and_redeploy.sh audio_player
```

不传参时仍按原有顺序更新全部模块。

### audio_player 的二进制来源（install 目录）

为了支持只发布二进制而不包含源代码，`libs/audio_player` 新增了 `install` 目录（可提交到仓库）。
当 `audio_player` 使用 `deploy` 流程时，脚本会从 `libs/audio_player/install/audio_player_service` 读取二进制并部署；若不存在则会触发本地构建。发布方可将编译好的二进制放到该 `install` 目录并提交，以便远端拉取后直接部署。

示例：发布者在仓库中更新 `libs/audio_player/install/audio_player_service`，然后运行：

```bash
./scripts/update_and_redeploy.sh audio_player
```

这样部署端无需源代码即可直接部署二进制。

## 说明

1. `EyeEngine` 的实际路径是 `modules/eyeEngine`，不是 `libs/EyeEngine`。
2. `audio_player` 对应的脚本文件名是 [manage_audio_player_service.sh](../../libs/audio_player/scripts/manage_audio_player_service.sh)，不是统一的 `manage_service.sh`。
3. `serial` 对应的脚本文件名是 [manage_serial_service.sh](../../libs/serial/scripts/manage_serial_service.sh)。
4. 脚本会在仓库更新完成后，依次调用各模块的 `redeploy` 命令完成重新编译和服务重启。
