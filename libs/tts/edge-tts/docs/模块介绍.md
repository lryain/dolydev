# libs/tts/edge-tts 模块介绍

## 这个模块是做什么的

`libs/tts/edge-tts` 是多利机器人的文本转语音模块。它负责把一段文本合成为语音音频文件，再交给音频播放系统播出。

## 小白可以怎么理解它

如果机器人“会说话”，这里负责的是“生成声音文件”，不是“实际播放喇叭”。

也就是说：

- `edge-tts` 负责把字变成音频
- `audio_player` 负责把音频播出来

## 它主要负责什么

- 调用 Edge TTS 语音合成能力
- 提供多语音人声选择
- 根据请求参数生成音频文件
- 为上层模块提供统一 TTS 服务接口

从现有文档可以确认，这个模块至少维护了语音列表与运维文档，说明它是一个可配置、可部署的服务模块。

## 输入与输出

### 输入

- 待合成文本
- 发音人、音调、语速、音量等参数

### 输出

- 生成的音频文件（如 wav、mp3）
- 可供播放器使用的音频路径和元数据

## 它依赖谁、又被谁依赖

### 它依赖谁

- Edge TTS 相关运行环境
- 本地存储和服务运行脚本

### 谁会用它

- `modules/doly` 中的 TTS 管理逻辑
- `libs/widgets` 中需要报时或提示语音的场景
- 最终通常会配合 `libs/audio_player` 完成播报

## 一个典型调用链

1. 主控或业务模块发起一段文本播报请求
2. `edge-tts` 生成音频文件
3. 返回文件路径给上层
4. 上层再调用 `audio_player` 进行播放

## 什么时候你应该看这个模块

- 文字能拿到，但没有生成语音文件
- 想切换发音人或调整说话风格
- 机器人播报内容发音不自然，想替换语音方案
