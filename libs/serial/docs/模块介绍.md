# libs/serial 模块介绍

## 这个模块是做什么的

`libs/serial` 是多利机器人和外部串口设备之间的通信模块。当前文档显示，它重点负责监听串口上的离线唤醒/离线指令模块，把串口收到的命令解析成系统可以处理的事件。

## 小白可以怎么理解它

这个模块就像机器人和某个外部小控制板之间的“翻译官”。

外部模块通过串口发来一串字节，`serial` 把它翻译成：

- 唤醒
- 高兴
- 音乐
- 开灯
- 关灯
- 聊天

然后再交给系统其它模块继续处理。

## 它主要负责什么

- 监听串口数据
- 解析约定好的命令字节
- 将串口命令发布为系统事件或命令
- 承担离线唤醒/离线按键类输入通道

现有文档已经明确说明了一些典型指令，例如：

- `0x01`：唤醒
- `0x02`：高兴
- `0x03`：音乐
- `0x04`：开灯
- `0x05`：关灯
- `0x07`：聊天

## 输入与输出

### 输入

- 串口字节流
- 串口协议配置

### 输出

- 解析后的命令事件
- 供主控或事件总线消费的系统动作触发

## 它依赖谁、又被谁依赖

### 它依赖谁

- 串口硬件
- 串口协议配置

### 谁会用它

- `modules/doly`：主控通过事件总线消费串口命令
- 其它需要离线输入触发的系统模块

## 一个典型调用链

1. 外部串口模块识别到用户离线指令
2. 串口发出一个命令字节
3. `libs/serial` 读取并解析这个字节
4. 将其转换成系统内部事件
5. 主控再决定执行哪种行为或动画

## 什么时候你应该看这个模块

- 离线唤醒无效
- 串口硬件有输入，但机器人没反应
- 新增了一条串口命令却没有映射成功

这是机器人“接收外部串口输入事件”的入口模块，偏输入链路，而不是动作执行链路。
