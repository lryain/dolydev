# libs/Doly 模块介绍

## 这个模块是做什么的

`libs/Doly` 是多利机器人最靠近硬件的一层 C++ 基础库。它把机械臂、底盘、边缘传感器、眼睛 LCD、LED、舵机、声音、ToF、触摸、TTS 等能力统一封装成一组稳定的控制接口，供上层模块和 SDK 调用。

这类模块通常不直接面向终端用户，而是给上层系统提供“能安全调用硬件”的能力。

## 小白可以怎么理解它

如果把整个机器人看作一台电脑：

- `modules/doly` 像总调度程序
- `modules/animation_system` 像动作编排器
- `SDK` 像对外开放的开发接口
- `libs/Doly` 就像机器人自己的“驱动层 + 硬件控制库”

没有这一层，上层模块就要自己直接处理 GPIO、LCD、传感器、舵机等细节，系统会很难维护。

## 它主要负责什么

- 封装底盘、机械臂、舵机、LCD、LED、触摸、ToF、IMU 等硬件控制接口
- 统一事件监听模型，例如完成事件、错误事件、状态变化事件
- 为 SDK 提供稳定的 C++ 头文件接口
- 降低上层模块直接操作硬件的复杂度

从 `include/` 目录可以直接看到它覆盖的能力范围，例如：

- `ArmControl.h`
- `DriveControl.h`
- `EyeControl.h`
- `LcdControl.h`
- `LedControl.h`
- `TofControl.h`
- `TouchControl.h`
- `TtsControl.h`

## 输入与输出

### 输入

- 上层模块传入的控制命令，例如移动、转向、抬手、设置灯光、播放声音
- 传感器初始化参数、校准参数、监听器回调对象

### 输出

- 对底层硬件的实际控制动作
- 设备状态、完成事件、错误事件、传感器数据

## 它依赖谁、又被谁依赖

### 它更靠近谁

- 更靠近 Linux 设备、GPIO、I2C、LCD、传感器和电机驱动

### 谁会用它

- `SDK` 会把这里的能力包装成更适合开发者使用的接口
- `modules/eyeEngine` 会直接或间接复用其中的 LCD 能力
- `libs/widgets` 会复用眼屏相关能力显示时钟和定时器
- 其他 C++ 服务模块也可能直接链接这里的静态库/动态库

## 什么时候你应该看这个模块

- 某个硬件完全不工作，例如 LCD、LED、机械臂、底盘没有响应
- 上层逻辑没问题，但设备执行结果异常
- 你要增加一个新的底层硬件能力，或者修改现有硬件控制参数

## 建议先看哪些文件

- `include/`：先看公开接口，理解模块能力边界
- `lib/` 和 `libs/`：看编译产物和第三方依赖
- `SDK/include/`：如果你想知道上层是怎么把这些能力对外暴露的，可以对照一起看
