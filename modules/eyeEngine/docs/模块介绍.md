# modules/eyeEngine 模块介绍

## 这个模块是做什么的

`modules/eyeEngine` 是多利机器人的眼睛动画引擎，负责把虹膜、眼睑、背景、序列帧等视觉元素渲染到左右 LCD 屏幕上，让机器人有“眼神”和“表情”。

## 小白可以怎么理解它

如果你觉得多利“像活的一样”，很大一部分来自这个模块。

它不仅能显示静态眼睛，还能：

- 眨眼
- 看左看右
- 切换情绪表情
- 播放 `.seq` 动画
- 配合上层系统做注视、提示和互动反馈

## 它主要负责什么

- 多层图像合成：虹膜、眼睑、背景
- 眼睛表情渲染与动画播放
- 程序化控制虹膜位置、颜色和主题
- 与 `animation_system` 的接口集成
- 与 `libs/widgets` 协调 LCD 使用权，避免同屏冲突

当前 README 已明确说明它支持：

- 表情系统
- 自动眨眼
- `.seq` 动画播放
- 与 `animation_system` 的无缝集成

## 输入与输出

### 输入

- 表情名称、动画名称、虹膜颜色、主题、位置
- 来自动画系统或主控模块的眼部表现命令
- Widget 的 LCD 抢占/释放事件

### 输出

- 240x240 左右眼 LCD 画面
- 眼睛动画播放状态
- 与上层服务协作时的状态事件

## 它依赖谁、又被谁依赖

### 它依赖谁

- LCD 驱动能力
- 眼睛资源文件（背景、虹膜、眼睑、`.seq` 动画）

### 谁会调用它

- `modules/animation_system`
- `modules/doly`
- 测试和演示脚本

### 它要和谁协调

- `libs/widgets`：两者都要使用眼屏 LCD，因此需要互斥显示

## 一个典型调用链

1. 主控或动画系统要求播放某个眼睛表现
2. `eyeEngine` 读取资源或根据参数生成状态
3. 渲染器合成左右眼最终画面
4. 通过 LCD 驱动写入屏幕
5. 若 Widget 需要显示，`eyeEngine` 暂停 LCD 使用，之后再恢复

## 什么时候你应该看这个模块

- 眼睛不显示或显示异常
- `.seq` 动画无法播放
- Widget 和眼睛显示发生抢屏
- 你要新增新的眼睛主题、表情或序列动画能力
